                          INTRODUCTION
                          ------------
  
0   PASCS IS A (SEMI-)COMPILER IMPLEMENTING PASCAL-S (A SUBSET OF THE 
 PROGRAMMING LANGUAGE PASCAL) ON THE HP 2100/21MX COMPUTER SYSTEMS, UN- 
 DER THE RTE-III OPERATING SYSTEM.
  
 PASCS TRANSLATES PASCAL-S PROGRAMS INTO (PSEUDO-)RELOCATABLE BINARY
 CODE. THIS CODE MUST BE LOADED ALONG WITH A RUN-TIME LIBRARY (PARUN) WHICH 
 PROVIDES SUPPORT FOR THE EXECUTION.
  
 BOTH PASCS AND PARUN HAVE BEEN CODED IN HP-ASSEMBLER (SOURCE PROGRAMS
 PACCC AND PADDD).
  
 THE HARDWARE REQUIREMENTS FOR THE PASCAL-S COMPILER ARE AN HP RTE SYSTEM 
 HAVING A PARTITION OF AT LEAST 12 PAGES. TO INSTALL PASCAL-S, FIRST GET
 PACCC (ALTER FIRST THE LINES 8-14 OF IT, IF YOU NEED LARGER TABLES), AS- 
 SEMBLE AND THEN LOAD IT, ASKING FOR A PARTITION SIZE SOMEWHAT LARGER 
 THAN IS STRICTLY NEEDED (THE EXTRA SPACE IS USED BY PASCS FOR ITS WORKING
 STACK). SAVE THE ABSOLUTE PROGRAM PASCS. THEN ASSEMBLE PADDD. ITS CONTROL
 STATEMENT IS   ASMB,R,L,T,F   AND SHOULD BE CHANGED IF THE TARGET MACHINE
 DOES NOT POSSESS THE FLOATING POINT HARDWARE OPTION. THE RELOCATABLE 
 TEXT OBTAINED FROM THE ASSEMBLY OF PADDD SHOULD BE SAVED IN PARUN. 
  
 TO RUN THE COMPILER, A TYPE 5 FILE, HAVING THE NAME PASBIN, MUST BE PRELIMI- 
 NARLY CREATED. PASCS TRANSLATES PASCAL-S PROGRAMS INTO CODE FOR AN HYPOTHE-
 TICAL STACK MACHINE. THE CODE HAS THE FORMAT  OP-CODE (0 TO 63)  FOLLOWED
 BY ZERO, ONE, OR TWO OPERANDS. PASCS PRODUCES IN PASBIN A SEQUENCE OF
 SUBROUTINE CALLS OF THE FORM   JSB SUBXX  (WHERE XX IS 00 TO 63), FOLLOWED 
 EACH BY ZERO, ONE, OR TWO OPERANDS. THE SUBROUTINES SUB00, SUB01,...,SUB63,
 EACH ONE OF WHICH EFFECTIVELY INTERPRETS ONE OF THE HYPOTHETICAL INSTRUCTIONS
 OF THE PSEUDO-CODE, ARE CONTAINED IN THE RUN-TIME LIBRARY PARUN. 
 TWO MORE SUBROUTINES (SUB64 AND SUB65) ARE USED TO HANDLE CALLS TO 
 EXTERNAL SUBPROGRAMS. THE EXTERNAL SUBPROGRAMS ARE GIVEN REFERENCE 
 NUMBERS FROM 66 ONWARDS. 
  
    THE PASCAL-S COMPILER IS SCHEDULED BY THE OPERATOR BY A REQUEST OF THE
 FORM 
                :RUN,PASCS [,P1,P2,P3,P4,P5]
  
 WHERE THE OPTIONAL PARAMETERS HAVE THE FOLLOWING MEANING:
  
    P1 = LOGICAL UNIT OF COMPILER INPUT DEVICE (STANDARD IS 1; MUST BE SET
         TO 2 FOR INPUT FROM LS TRACKS).
  
    P2 = LOGICAL UNIT OF COMPILER LIST DEVICE (STANDARD IS 6).
  
    P3 = LOGICAL UNIT OF INPUT FOR THE COMPILED PROGRAM (STANDARD IS 1; 
         MUST BE SET TO 2 FOR INPUT FROM LS TRACKS).
  
    P4 = LOGICAL UNIT OF OUTPUT FOR THE COMPILED PROGRAM (STANDARD IS 6). 
  
    P5 = MAXIMUM NUMBER OF OUTPUT LINES TO BE PRODUCED DURING EXECUTION 
         OF THE COMPILED PROGRAM (STANDARD IS 300).

 IT SHOULD BE NOTICED THAT NORMAL COMPILATION LISTING CONTAINS, AT THE
 BEGINNING OF EACH PROGRAM LINE, A NUMBER, INDICATING THE POSITION, IN
 THE CODE PRODUCED IN PASBIN, WHERE THE TRANSLATION OF THE LINE BEGINS. 
 THIS IS USEFUL WHEN INTERPRETING A POST-MORTEM DUMP. MOREOVER, IF THE
 PROGRAM NAME IS   TEST0  , A FULL LISTING IS ALSO GIVEN OF ALL THE 
 (PSEUDO-)CODE PRODUCED, AND OF THE VARIOUS TABLES PRODUCED DURING THE
 COMPILATION. THIS IS NORMALLY USEFUL ONLY TO THE IMPLEMENTER OF THE
 SYSTEM, TO EXAMINE ITS INTERNAL OPERATION.  ALSO NOTICE THAT, WHEN THE 
 OUTPUT UNIT USED IS A LINE PRINTER, THE FIRST CHARACTER OF EACH OUTPUT 
 LINE IS USED FOR CARRIAGE CONTROL (BLANK PRODUCES SINGLE SPACING, 0
 PRODUCES DOUBLE SPACING, AND 1 SKIPS TO THE TOP OF NEXT PAGE). 
  
  
    FOR DETAILS ABOUT PASCAL AND PASCAL-S THE LITERATURE REFERENCES 
 GIVEN IN THE APPROPRIATE SECTION SHOULD BE CONSULTED.
  
 PASCAL-S DIFFERS FROM FULL PASCAL IN THE FOLLOWING ASPECTS:
  
   - SCALAR AND SUBRANGE TYPES ARE OMITTED; THERE ARE NO POINTER TYPES; 
  
   - SET AND FILE STRUCTURES ARE OMITTED, EXCEPT FOR TWO STANDARD FILES 
     (INPUT AND OUTPUT), WHICH ARE PREDECLARED, BUT MUST BE LISTED IN THE 
     PROGRAM HEADING IF REQUIRED (OUTPUT IS ALWAYS REQUIRED); 
  
   - VARIANTS IN RECORDS ARE NOT ALLOWED; 
  
   - THERE ARE NO PACKING OPTIONS;
  
   - WITH AND GO TO STATEMENTS ARE OMITTED; THERE ARE NO LABELS;
  
   - PROCEDURES AND FUNCTIONS CANNOT BE USED AS PARAMETERS; 
  
   - ONLY THE FOLLOWING STANDARD OBJECTS ARE AVAILABLE: 
  
       CONSTANTS: TRUE, FALSE;
       TYPES: INTEGER, REAL, BOOLEAN, CHAR; 
       PROCEDURES: READ, READLN, WRITE, WRITELN;
       FUNCTIONS: ABS, SQR, ODD, CHR, ORD, SUCC, PRED, ROUND, TRUNC,
                  SIN, COS, EXP, LN, SQRT, ARCTAN, EOF, EOLN. 
  
  
 A FEW ADDITIONS HAVE BEEN INTRODUCED INTO THE STANDARD PASCAL LANGUAGE:
  
   - THERE IS A STANDARD FUNCTION RANDOM (FUNCTION RANDOM:REAL;) WHICH
     GENERATES PSEUDORANDOM NUMBERS IN THE INTERVAL [0,1);
  
   - THERE IS A STANDARD PROCEDURE HALT, WHICH TERMINATES EXECUTION;
  
   - A FACTOR MAY ALSO HAVE THE FORM
         TYPE-IDENTIFIER "("  EXPRESSION  ")" 
     AS FOR EXAMPLE  REAL(2+1). THE EXPRESSION, AFTER EVALUATION, IS
     CONSIDERED AS HAVING THE TYPE   TYPE-IDENTIFIER  , WITH NO CON-
     VERSION BEING APPLIED.

   - IT IS POSSIBLE TO USE PROCEDURES AND FUNCTIONS (WITH NO LOCAL VARIA- 
     BLES) WRITTEN SEPARATELY IN FORTRAN OR ASSEMBLER. IN THE PASCAL PRO- 
     GRAM THEY ARE DECLARED WITH A DUMMY BODY (I. E. THE KEYWORD FORTRAN).
     THE BODIES ARE COMPILED AS FOLLOWS:
       EXT PROC  JSB PROC  DEF *+N+1  DEF PAR1 ... DEF PARN   , 
     WHERE N IS THE TOTAL NUMBER OF PARAMETERS PASSED TO THE
     ROUTINE. THE EXTERNAL ROUTINE SHOULD FETCH ITS PARAMETERS
     IN THE FORTRAN STYLE, THAT IS VIA THE   .ENTR   ROUTINE. THERE ARE 
     SEVERAL INCOMPATIBILITIES TO BE TAKEN INTO ACCOUNT.
     IT SHOULD FIRST BE NOTICED THAT IN PASCAL-S OBJECTS OF THE BASIC 
     TYPES (CHAR,BOOLEAN,INTEGER AND REAL) ALL OCCUPY TWO CONSECUTIVE 
     WORDS. CHARS ARE PLACED IN THE RIGHTMOST BYTE OF THE FIRST WORD. 
     BOOLEANS AND INTEGERS USE ONLY THE FIRST WORD. BOOLEAN VALUES ARE
     CONSIDERED TRUE OR FALSE ACCORDING TO WHETHER THE LEFTMOST BIT IS
     ON OR OFF. 
     WHEN TRANSMITTING SINGLE ELEMENTS OF THE BASIC TYPES TO EXTERNAL 
     ROUTINES, NO PROBLEM SHOULD ARISE. HOWEVER, WHEN TRANSMITTING AR-
     RAYS, THE ROUTINE SHOULD SEE THEM AS ARRAYS OF DOUBLE WORDS (I. E. 
     AS REAL ARRAYS), FROM THE POINT OF VIEW OF CORRECT INDEXING. BUT OF
     COURSE IT SHOULD ONLY OPERATE ON THE FIRST WORD, ACCORDING TO ITS
     TYPE (CHARACTER,INTEGER OR BOOLEAN). 
     ANOTHER INCOMPATIBILITY IS PROVIDED BY THE WAY ARRAYS ARE STORED.
     IN FORTRAN THE LAST SUBSCRIPT IS THE ONE THAT RUNS FASTEST (I. E.
     TWO-DIMENSIONAL ARRAYS ARE STORED COLUMNWISE). IN PASCAL THINGS
     ARE REVERSED, I. E. THE LEADING SUBSCRIPT RUNS THE FASTEST.
  
     A SIMPLE EXAMPLE OF EXTERNAL CALL IS GIVEN HERE (NORM OF A MATRIX).
  
 PROGRAM TEST0(OUTPUT);CONST R=4;S=5;TYPE MATRIX=ARRAY[1..R,1..S]OF INTEGER;
         VAR I,J,K:INTEGER; B:MATRIX; 
 FUNCTION NORM(VAR A:MATRIX;M,N:INTEGER):INTEGER;FORTRAN; 
 BEGIN K:=0;FOR I:=1 TO R DO FOR J:=1 TO S DO BEGIN K:=K+1;B[I,J]:=K END; 
       WRITELN(' ',NORM(B,R,S)) 
 END .
  
          FUNCTION NORM(A,M,N)
          DIMENSION A(M,N)
          EQUIVALENCE (Z,K) 
          NORM=0
          DO 1 I=1,M
          DO 1 J=1,N
          Z=A(I,J)
    1     NORM=NORM+IABS(K) 
          RETURN
          END 
  
     THE PROGRAM SHOULD PRODUCE THE RESULT     210     .

   NOTES. 
   ------ 
  
       ROUND BOXES DENOTE SYMBOLS OF THE LANGUAGE PASCAL, RECTANGULAR 
  BOXES DENOTE SYNTACTIC CONSTRUCTS REPRESENTED BY DIAGRAMS. SEPARATORS 
  ARE BLANKS, END OF LINES, AND COMMENTS. A COMMENT IS AN ARBITRARY SE- 
  QUENCE OF CHARACTERS ENCLOSED WITHIN A PAIR OF COMMENT BRACKETS  (* 
  AND  *). AT LEAST ONE SEPARATOR MUST OCCUR BETWEEN CONSECUTIVE
  IDENTIFIERS, NUMBERS, AND WORD-SYMBOLS (SUCH AS BEGIN, END). SEPA-
  RATORS MAY BE INSERTED BETWEEN ANY TWO SYMBOLS; HOWEVER, NO SEPARATORS
  MUST OCCUR WITHIN NUMBERS AND IDENTIFIERS.
       IN A SYNTAX DIAGRAM, THE OCCURRENCE OF THE NON-QUALIFIED WORD "IDEN- 
  TIFIER" IMPLIES THAT AT THIS POINT AN ARBITRARY, NEW IDENTIFIER MAY BE
  CHOSEN. THIS IDENTIFIER THEREBY BECOMES A CONSTANT-, TYPE-, VARIABLE-,
  FIELD-, FUNCTION-, OR PROCEDURE- IDENTIFIER.
       THE COLON MAY BE REPRESENTED AS .. OR AS :  . STRINGS (I.E. SEQUEN-
  CES OF CHARACTERS ENCLOSED BETWEEN ' AND ' ) MAY BE USED ONLY AS ARGU-
  MENTS FOR WRITE AND WRITELN. TO REPRESENT A QUOTE AS A CHARACTER CONS-
  TANT OR WITHIN A STRING, ONE MUST WRITE IT TWICE. IDENTIFIERS DENOTING
  DISTINCT OBJECTS MUST DIFFER OVER THEIR FIRST TEN CHARACTERS. 
       ALL STANDARD OBJECTS (CONSTANTS, TYPES, PROCEDURES AND FUNCTIONS,
  LISTED BEFORE THE SYNTAX DIAGRAMS) ARE ASSUMED AS DECLARED IN A SCOPE 
  SURROUNDING THE PROGRAM; NO CONFLICT ARISES FROM A DECLARATION REDEFINING 
  THE SAME IDENTIFIER WITHIN THE PROGRAM. 
       THE FUNCTIONS SUCC, PRED AND ORD ARE APPLICABLE TO ARGUMENTS OF TYPE 
  CHAR, INTEGER AND BOOLEAN.

  WITH THE FOLLOWING MESSAGE:

  WHERE XXX IS THE ERROR POSITION IN THE CODE, AND YYYYY MAY BE ONE 
  OF THE FOLLOWING: 
  
    UNDEFINED CASE      SELECTOR VALUE OF A CASE STATEMENT IS OUT OF RANGE. 
    DIVISION BY 0       ATTEMPT TO DIVIDE BY ZERO.
    INVALID INDEX       ARRAY INDEX OUT OF BOUNDS.
    STORAGE OVERFLOW    OVERFLOW OF THE RUN-TIME STACK. 
    TOO MUCH OUTPUT     THE MAXIMUM NUMBER OF OUTPUT LINES HAS BEEN EXCEEDED. 
    LINE TOO LONG       MORE THAN 136 CHARACTERS IN AN OUTPUT LINE. 
    READING PAST END OF FILE
                        ATTEMPT TO READ BEYOND THE END OF THE INPUT FILE. 
    DIGIT NOT FOUND     A DIGIT WAS NOT FOUND WHERE IT WAS EXPECTED, WHILE
                        READING A REAL NUMBER.
    NUMBER OVERFLOW     OVERFLOW WHILE READING A NUMBER (INTEGERS SHOULD BE 
                        IN THE RANGE FROM -32768 TO 32767, REALS SHOULD BE
                        IN THE RANGE FROM  -2**127 TO (2**127)*(1-2**(-23)) ).
    ERROR (SEE ABOVE)   PRECEDED BY A MESSAGE OS THE FORM 
                           PNAME : AA BB
                        WHERE  PNAME IS THE NAME OF THE PROGRAM RUNNING,
                        AA=00  BB=OF   OVERFLOW IN INTEGER OPERATION
                                       (+,-,*,ROUND,TRUNC,SQR). 
                           01     OF   OVERFLOW IN REAL OPERATION 
                                       (+,-,*,/,SQR). 
                           02     UN   LOGARITHM OF A NUMBER <=0.

       THE PROGRAM READ IN FROM A DISC FILE IS INCOMPLETE, OR A CONTI-
       NUATION TEXT WAS EXPECTED FROM A PERIPHERAL DEVICE, BUT IT HAS 
       BEEN ANSWERED THAT NONE IS FOLLOWING.

       THE WORKING STACK OF THE COMPILER HAS OVERFLOWED.
